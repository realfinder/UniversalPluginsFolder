low_res_color_deblocking

The following are frames from an old “1995” very badly encoded TV show. The encoded frame size is 352x240. 
They all use the following calls which produces a luma smoothed image in the top left,  a luma and chroma smoothed image in the top right
The difference image in the bottom left, and the differences multiplied by six in the bottom right. 

a=SmoothD2(quant=18, num_shift=2, Matrix=8, Qtype=1, ZW=0, ZWce=1, ZWlmDark=255, ZWlmBright=140)
a.SmoothD2c(quant=18, num_shift=3, Matrix=5, Qtype=1, ZW=0, cWarpToLuma=5, downSize=1)

ShowDiffMult(a, mult=6)

















The following example shows how the use of cWarpToLuma affects the image.  
The left image does not use it and the right image uses it.  
The difference between the bottom right quadrants of the image above this paragraph and the image below shows the color deblocking that
SmoothD2 is doing. 

a=SmoothD2(quant=18, num_shift=2, Matrix=8, Qtype=1, ZW=0, ZWce=1, ZWlmDark=255, ZWlmBright=140)
leftImage  =a.SmoothD2c(quant=18, num_shift=3, Matrix=5, Qtype=1, ZW=0, cWarpToLuma=0, downSize=1)
rightImage=a.SmoothD2c(quant=18, num_shift=3, Matrix=5, Qtype=1, ZW=0, cWarpToLuma=5, downSize=1)
rightImage
return(ShowDiffMult(leftImage, mult=6))
